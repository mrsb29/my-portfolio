    Stripe Recurring Payment Integration with Laravel and Angular

    • Why Stripe is best for Recurring Payments?

      ● Developer-Friendly: Stripe provides a well-documented API.

      ● Secure Transactions: It ensures PCI compliance.

      ● Multiple Payment Methods: Supports credit cards, wallets, and bank transfers.

      ● Automated Billing & Invoicing:Handles retries and notifications.

    • Setting Up Stripe:
      Step 1: Create a Stripe Account & and get key's:

      To get keys Navigate to Developers > API Keys to get your Publishable Key and Secret Key.
      Step 2: Install Stripe PHP SDK & Configure it in Laravel:

      Using the following command to install the Stripe SDK: composer require stripe/stripe-php,then configuring stripe to adding key in.envfile

    • Create a Subscription Controller & Define API Route:

      Create controller for handeling payment logic and define a api route for that.

      code-snippet
      ?php
      namespace App\Http\Controllers;
      use Illuminate\Http\Request;
      use Stripe\Stripe;
      use Stripe\Customer;
      use Stripe\Subscription;
      use Stripe\PaymentMethod;
      class SubscriptionController extends Controller
      {
       public function createSubscription(Request $request)
      {
       Stripe::setApiKey(env('STRIPE_SECRET'));
      try {
        $customer = Customer::create([
         'email' => $request->email,
          'name' => $request->name,
         'payment_method' => $request->paymentMethodId,
          'invoice_settings' => ['default_payment_method' => $request->paymentMethodId]
       ]);
        $subscription = Subscription::create([
         'customer' => $customer->id,
         'items' => [['price' => $request->priceId]],
         'expand' => ['latest_invoice.payment_intent'],
        ]);
        return response()->json(['subscriptionId' => $subscription->id, 'status' => $subscription->status]);
      } catch (\Exception $e) {
         return response()->json(['error' => $e->getMessage()], 400);
          }
        }
      }

    • Setting Up Stripe:
      Step 1:Install Stripe.js in Angular

      Run the command: npm install @stripe/stripe-js And Add Stripe.js to angular.json
      Step 2: Create Payment Service in Angular:

      A payment service in Angular is essential for handling communication with Stripe and the Laravel backend.
      Step 3: Implement Payment in Angular Component:

      This step involves integrating the payment service into the Angular component to handle user interactions, initialize the Stripe elements, and send payment requests to the backend.
      Step 4: Add Payment Form:

      A simple HTML form is used to collect the user's card details.
      Step 5: Handle Webhooks in Laravel:

      Stripe sends webhook events for subscription updates, failed payments, and cancellations.

    • Conclusion:

      This guide covered the integration of Stripe recurring payments using Laravel and Angular. I implemented backend logic in Laravel to process subscriptions and built a dynamic payment UI in Angular. This approach ensures a secure, scalable, and user-friendly subscription management system.

    Custom Multitenancy with Dynamic Databases in Laravel and Angular .

    • Multitenancy is a crucial architecture for SaaS applications, allowing multiple clients to use the same application while maintaining isolated data. In this blog, we will explore a custom multitenancy implementation using Laravel for the backend and Angular for the frontend. This approach ensures seamless client management, dynamic database provisioning, and secure data access.

    • Multitenancy Architecture Overview

      ● Database for Client Credentials: A central database stores client details such as client code, database name, and business information.

      ● Dynamic Database Creation: When a user registers or purchases the software, a new database is automatically created for them, and its credentials are stored in the client credentials database.

      ● Login & Client Identification: When users log in, their credentials are validated, and the system determines which client they belong to. The client_id is returned in the response.

      ● Secure Frontend Client Storage: The frontend stores client_id securely using crypto-js and sends it in every request as a global header.

      ● Middleware-Based Database Switching: A custom Laravel middleware intercepts every request, retrieves the client's database credentials, and dynamically switches the database connection.

      ● Service-Based Database Credential Fetching: A dedicated Laravel service fetches the database credentials and ensures each client accesses only their data.

    From Code to Cloud: My Simple Guide to Development and Deployment

    • In today’s fast-paced software world, writing code is just one part of the journey. Once the development is done, the next crucial step is getting that code live – a process we call deployment. In this blog, I’ll walk you through a simple overview of both development and deployment, and how they go hand in hand.

    • 1. Development – Building the Application

      Development is the phase where we write code to build our application. It involves:

      ● Planning the features – What the app will do.

      ● Designing the structure – How it’s going to look and function.

      ● Writing the code – Using languages like PHP, JavaScript, or frameworks like Laravel or Angular.

      ● Testing – Making sure everything works as expected before release.

    • 2. Deployment – Going Live

      Once the app is ready, deployment is the step where we move the code from our local machine to a live server. Common steps include:

      ● Pushing code to GitHub or GitLab

      ● Setting up a server.

      ● Using CI/CD – Tools like GitHub Actions or Bitbucket Pipelines help automate the deployment process.

      ● Environment configuration – Setting .env files, database connections, caching, etc.

    • For Laravel projects, we often run these commands on the server:

      Commands
      composer install
      php artisan migrate
      php artisan config:cache

    • Development and deployment are two sides of the same coin. While development is about building, deployment is about delivering. With the right practices in place, you can ensure a smooth path from local to live.
